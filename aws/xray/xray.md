# AWS Service Cost Research: AWS X-Ray

> **Status:** ✅ Research Complete
> **Last Updated:** July 2026

---

## 1. Service Overview
AWS X-Ray helps developers analyze and debug production, distributed applications, such as those built using a microservices architecture. X-Ray collects trace data generated by incoming HTTP requests, downstream service calls, database queries, and AWS SDK calls across Lambda functions, ECS tasks, EC2 instances, and API Gateways. X-Ray builds a visual service map showing latency bottlenecks and error rates. X-Ray is billed per million trace segments ingested and retrieved.

---

## 2. Billing Mechanics
AWS X-Ray billing is calculated across two primary dimensions:
1.  **Traces Recorded (Data Ingestion):** Billed per 1 Million trace segments recorded by X-Ray daemon or agent ($5.00 per 1 Million traces).
2.  **Traces Retrieved / Scanned (Analytics):** Billed per 1 Million trace segments scanned or retrieved when querying the X-Ray console or API ($0.50 per 1 Million traces).

---

## 3. Key Cost Dimensions

### A. Core Tracing Rates (us-east-1)
*   **Traces Recorded (Ingestion):** **$5.00 per 1 Million traces** ($0.000005 per trace).
*   **Traces Retrieved / Scanned (Analytics):** **$0.50 per 1 Million traces** ($0.0000005 per trace).

### B. Math Example (High Volume Hazard)
If a microservice handles 100 Million requests/month with 100% X-Ray trace sampling enabled:
$$100\text{ Million traces} \times \$5.00 = \$500.00\text{ / month}$$

## 5. AWS Free Tier Coverage
*   **AWS X-Ray Free Tier:** Includes **100,000 free traces recorded** and **1,000,000 free traces retrieved** per month indefinitely for all AWS accounts.

---

## 6. Common Cost Hotspots & Pitfalls
*   **Setting 100% Sampling Rates on High-Volume APIs (The Trace Ingestion Spike):**
    *   Configuring X-Ray SDK sampling rules to record 100% of incoming HTTP requests on a high-throughput public API.
    *   *Math:* Ingesting 500 Million trace segments/month = **$2,500.00/month in X-Ray tracing fees!**

---

## 7. Actionable Cost Optimization Strategies
1.  **Enforce Targeted Sampling Rules (5% Fixed Rate + 100% Errors):**
    *   Configure **X-Ray Sampling Rules** in the console or SDK.
    *   Set rule: *Sample 5% (or 1%) of successful 2xx HTTP responses, but sample 100% of 4xx and 5xx error responses*.
    *   **The Savings:** Cuts X-Ray trace ingestion volume and billing by **90–95%** while retaining 100% error diagnostic visibility.
2.  **Exclude Health Check & Ping Endpoints:** Add sampling exclusion rules to bypass tracing `/healthz`, `/ping`, or load balancer health check URIs.
3.  **Use OpenTelemetry (ADOT) Collector:** Use **AWS Distro for OpenTelemetry (ADOT)** to filter and aggregate trace segments locally before emitting to X-Ray or CloudWatch.
