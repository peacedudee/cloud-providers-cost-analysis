# AWS Service Cost Research: Amazon Bedrock

> **Status:** ✅ Research Complete & Ground Truth Verified  
> **Last Updated:** July 2026

---

## 1. Service Overview
Amazon Bedrock is a fully managed, serverless service that provides access to leading foundation models (FMs) from Anthropic, Cohere, Meta, AI21 Labs, Stability AI, and Amazon via a unified API. Bedrock abstracts underlying GPU infrastructure, allowing developers to build generative AI applications securely. Bedrock is serverless, billing based on token consumption, provisioned throughput, batch inference, or custom model fine-tuning.

---

## 2. Billing Mechanics
Bedrock charges are billed monthly across four primary dimensions:
1. **On-Demand Token-Based Billing:** Billed per 1 Million tokens processed (separated into input prompt tokens and output generated tokens).
2. **Batch Inference (50% Discount):** Asynchronous processing for non-real-time workloads, providing a **50% discount** on standard token rates.
3. **Provisioned Throughput (Model Units):** Billed per Model Unit (MU) hour for dedicated capacity, with 1-month and 6-month commitment discounts.
4. **Guardrails & Knowledge Bases:** Surcharges for safety filtering ($1.00/1k requests) and vector storage index compute.

---

## 3. Key Cost Dimensions

### A. On-Demand Token Pricing (us-east-1 Examples)
* **Token Ratio:** Output tokens (text generated by the model) are **4x to 5x more expensive** than input prompt tokens because generation requires iterative compute.
* *Claude 3.5 Sonnet (High intelligence standard):*
  * *Input Tokens:* **$3.00 per Million tokens**.
  * *Output Tokens:* **$15.00 per Million tokens**.
* *Claude 3.5 Haiku (Speed and cost-optimized):*
  * *Input Tokens:* **$0.25 per Million tokens** (91% cheaper prompts).
  * *Output Tokens:* **$1.25 per Million tokens**.
* *Llama 3.1 8B (Open model standard):*
  * *Input Tokens:* **$0.22 per Million tokens**.
  * *Output Tokens:* **$0.22 per Million tokens**.

### B. Provisioned Throughput (Dedicated Model Units)
* Best for enterprise applications requiring guaranteed latency and throughput.
* Billed per MU-hour. Commitments (1-month or 6-month) offer up to a **50% discount**.
* *Warning:* Provisioned throughput runs continuously. An idle 1 MU Claude 3.5 Sonnet configuration incurs fixed hourly charges 24/7.

### C. Bedrock Guardrails
* **The Rate:** **$1.00 per 1,000 requests** evaluated for PII redaction, toxicity, or topic filtering.
* *Impact:* Adding Guardrails to lightweight prompts can exceed the cost of the LLM token invocation itself.

### D. Knowledge Bases & Vector Index Compute
* Knowledge Bases automatically integrate vector search for RAG workflows.
* Default deployment using Amazon OpenSearch Serverless incurs a minimum index compute charge of **$350.40/month** (2 OCUs baseline), representing a major hidden base charge for small vector datasets.

## 5. AWS Free Tier Coverage
* **Amazon Bedrock:** No free tier available. All model invocations generate immediate token billing.

## 6. Common Cost Hotspots & Pitfalls
* **Using High-Tier Models for Simple Tasks:** Routing simple classification or extraction queries to Claude 3.5 Sonnet ($3.00/M input) instead of Claude 3.5 Haiku ($0.25/M input).
* **Unbounded Prompt Engineering & Chat History:** Forwarding entire historical chat logs on every API call, multiplying input token consumption exponentially.
* **Default OpenSearch Serverless Vector Indexes:** Creating a Knowledge Base with default OpenSearch Serverless collections, incurring **$350.40/month** even for a 1 MB document repository.
* **Uncapped Token Generation:** Omitting `max_tokens` parameters, allowing models to generate runaway text output ($15.00/M output).

---

## 7. Actionable Cost Optimization Strategies
1. **Implement Model Cascading (Router Architecture):**
   * Route incoming prompts to **Claude 3.5 Haiku** or **Llama 3.1 8B** first.
   * Only escalate complex queries (failed validation or code debugging) to **Claude 3.5 Sonnet**.
   * **The Savings:** Slashes total token costs by **60%–80%**.
2. **Use Batch Inference for Asynchronous Workloads:**
   * Run offline batch jobs (e.g., document summarization or dataset processing) using **Bedrock Batch Inference**.
   * **The Savings:** Secures a direct **50% discount** on all token pricing.
3. **Use Aurora/RDS Vector Storage for Knowledge Bases:**
   * Select **Amazon Aurora (pgvector)** or existing RDS PostgreSQL instances for Knowledge Base storage instead of default OpenSearch Serverless.
   * **The Savings:** Bypasses the **$350.40/month** flat OCU baseline fee.
4. **Set Enforced `max_tokens` Limits:** Always define strict output token limits (e.g., `max_tokens: 500`) on model invocation payloads.
