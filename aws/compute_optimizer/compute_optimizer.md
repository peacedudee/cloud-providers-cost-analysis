> **Status:** ✅ Research Complete & Ground Truth Verified

## 1. Service Overview
AWS Compute Optimizer uses machine learning to analyze historical operational metrics and deliver right-sizing recommendations for your AWS workloads. It identifies over-provisioned resources (to cut costs) and under-provisioned resources (to eliminate performance bottlenecks). Compute Optimizer supports a wide range of AWS services, including:
*   Amazon EC2 instances and EC2 Auto Scaling groups
*   Amazon EBS volumes
*   Amazon ECS services on AWS Fargate
*   AWS Lambda functions
*   Amazon RDS database instances (MySQL, PostgreSQL, MariaDB, SQL Server, Oracle)
*   Amazon DynamoDB tables and global secondary indexes
*   Amazon ElastiCache clusters (Redis and Memcached)
*   Amazon MemoryDB for Redis clusters
*   Amazon OpenSearch Service clusters
*   Amazon SageMaker endpoints (ML inference)
*   AWS NAT Gateways

## 2. Billing Mechanics
1.  **Basic Recommendations (14-Day Lookback Window):** **100% Free ($0.00)**. No setup, analysis, or recommendation fees.
2.  **Enhanced Infrastructure Metrics (3-Month Lookback Window):** Billed hourly per monitored resource ($0.000336 per resource-hour = ~$0.25 per resource-month). This is useful for identifying seasonal or quarterly usage patterns.
3.  **No Savings Commissions:** AWS does not take a percentage of the savings generated by Compute Optimizer.

## 6. Common Cost Hotspots & Pitfalls
*   **Ignoring Over-Provisioned Recommendations:** Allowing resources tagged with `Over-provisioned` (e.g. running an `m5.4xlarge` EC2 instance when an `m5.large` handles peak load) to run unchanged, wasting 75% of compute spend.
*   **Enabling Enhanced Metrics Globally:** Enabling the 3-month lookback window ($0.25/resource-mo) on thousands of short-lived developer sandbox resources or ephemeral test containers.
*   **Assuming Recommendations are Automated:** Thinking Compute Optimizer automatically sizes resources down. Compute Optimizer only provides *reports*; manual action or IaC updates are required to capture the savings.

## 7. Actionable Cost Optimization Strategies
1.  **Export Organization-Wide Recommendations for FinOps Analysis:**
    *   Enable **AWS Compute Optimizer Organizational View** in your Management Account.
    *   Export right-sizing recommendations for all member accounts to an Amazon S3 bucket via the AWS CLI or console.
    *   Filter recommendations by `Finding = Overprovisioned` or `Finding = Underprovisioned`.
    *   **The Savings:** Provides an immediate roadmap to capture **30–60% compute cost reductions** across the entire organization.
2.  **Rightsize Newly Supported Database and Caching Layers:**
    *   **Amazon RDS:** Identify idle or over-provisioned DB instances. Migrate underutilized DBs to standard instance families or smaller storage configurations.
    *   **Amazon ElastiCache & MemoryDB:** Review MemoryDB and ElastiCache recommendations to shrink over-provisioned cache nodes or switch to Serverless caching models for highly variable workloads.
    *   **Amazon DynamoDB:** Transition tables with steady, predictable traffic from On-Demand (WRU/RRU) to Provisioned capacity (WCU/RCU) with Auto Scaling, or vice-versa for sparse workloads.
3.  **Optimize NAT Gateways and SageMaker Endpoints:**
    *   Review Compute Optimizer recommendations for **NAT Gateways** to identify completely idle gateways or those processing minimal traffic (which can be replaced by VPC Endpoint interfaces).
    *   Assess **SageMaker Endpoints** to scale down instance sizes for ML model inference hosting during off-peak testing hours.
4.  **Prioritize EC2 & EBS Upgrades:**
    *   Upgrade old EBS GP2 volumes to **GP3** based on recommendations (saving 20% flat with higher baseline IOPS).
    *   Review EC2 instance family migrations (e.g., migrating `m5` to Graviton `m6g` for 20% lower cost and up to 40% higher performance).
5.  **Use Basic Recommendations ($0.00) in Non-Production:** Keep 14-day Basic Recommendations active in dev/test accounts ($0.00) and enable paid Enhanced Metrics ($0.25/mo) only for production workloads with quarterly/seasonal cycles.
