# AWS Service Cost Research: AWS Compute Optimizer

> **Status:** ✅ Research Complete
> **Last Updated:** July 2026

---

## 1. Service Overview
AWS Compute Optimizer uses machine learning to analyze historical operational metrics and deliver right-sizing recommendations for your AWS workloads. It evaluates Amazon EC2 instances, EC2 Auto Scaling groups, Amazon EBS volumes, AWS Lambda functions, and Amazon ECS tasks on Fargate. Compute Optimizer identifies over-provisioned resources (to cut costs) and under-provisioned resources (to eliminate performance bottlenecks).

---

## 2. Billing Mechanics
1.  **Basic Recommendations (14-Day Lookback Window):** **100% Free ($0.00)**. No setup, analysis, or recommendation fees.
2.  **Enhanced Infrastructure Metrics (3-Month Lookback Window):** Billed hourly per monitored resource ($0.000336 per resource-hour = ~$0.25 per resource-month).
3.  **No Savings Commissions:** AWS does not take a percentage of the savings generated by Compute Optimizer.

## 6. Common Cost Hotspots & Pitfalls
*   **Ignoring Over-Provisioned Instance Recommendations:** Allowing EC2 fleets tagged with `Over-provisioned` (e.g. running `c5.4xlarge` when `c5.large` handles peak load) to run unchanged, wasting 75% of compute spend.
*   **Enabling Enhanced Metrics Across All Non-Prod Resources:** Enabling 3-month Enhanced Infrastructure Metrics ($0.25/resource-mo) on thousands of short-lived developer sandbox instances.

---

## 7. Actionable Cost Optimization Strategies
1.  **Export Organization-Wide Recommendations for FinOps Analysis:**
    *   Enable **AWS Compute Optimizer Organizational View** in your Management Account.
    *   Export right-sizing recommendations to an Amazon S3 bucket via the AWS CLI or console.
    *   Filter recommendations by `Finding = Overprovisioned`.
    *   **The Savings:** Provides an immediate roadmap to capture **30–60% compute cost reductions**.
2.  **Prioritize EC2 & EBS Right-Sizing:**
    *   Review EBS volume recommendations: Compute Optimizer identifies GP2 volumes that can be upgraded to **GP3** (saving 20% instantly with higher IOPS).
    *   Review EC2 instance family migrations (e.g. migrating `m5` to Graviton `m6g` for 20% lower cost and 40% higher performance).
3.  **Use Basic Recommendations ($0.00) in Non-Production:** Keep 14-day Basic Recommendations active in dev/test accounts ($0.00) and enable Enhanced Metrics ($0.25/mo) only for mission-critical production workloads with quarterly seasonality.
