# AWS Service Cost Research: Amazon S3 (Simple Storage Service)

> **Status:** ✅ Research Complete
> **Last Updated:** July 2026

---

## 1. Service Overview
Amazon S3 is a highly durable, scalable object storage service. It is a foundational service used across almost all cloud applications. While storage unit pricing appears very cheap, S3 bills on multiple concurrent dimensions, making it a common source of unexpected billing spikes if request volumes, retrieval patterns, or lifecycle transitions are not optimized.

---

## 2. Billing Mechanics
S3 billing is calculated on a monthly cycle based on the following primary dimensions:
1.  **Storage Volume:** The amount of data stored (measured in GB-month) across different storage classes.
2.  **Requests & Data Retrievals:** The number of API calls made (PUT, GET, LIST, etc.) and the volume of data read (GB) from infrequent or archival classes.
3.  **Data Transfer:** Outbound network egress from S3 to the internet or other AWS regions.
4.  **Management & Replication Features:** Additional charges for object tagging, replication tracking, inventory reporting, Storage Lens, and Object Lambda processing.

---

## 3. Key Cost Dimensions

### A. Storage Classes (us-east-1)
*   **S3 Standard (Hot):** Billed at **$0.023 per GB-month** (first 50 TB). No retrieval fees, no minimum storage duration. Ideal for active data.
*   **S3 Intelligent-Tiering:** Automatically moves data between two access tiers (Frequent and Infrequent) to optimize costs. Billed at standard/IA rates plus a flat monitoring fee of **$0.0025 per 1,000 objects** per month. No retrieval fees.
*   **S3 Standard-IA (Infrequent Access):** Billed at **$0.0125 per GB-month**. Billed a retrieval fee of **$0.01 per GB** accessed. Has a **30-day minimum storage duration** and a **128 KB minimum billable object size**.
*   **S3 One Zone-IA:** Same as Standard-IA but stored in a single Availability Zone (20% cheaper storage: **$0.01 per GB-month**). Billed at $0.01/GB retrieved.
*   **S3 Glacier Instant Retrieval:** Billed at **$0.004 per GB-month**. Data is retrieved in milliseconds. Retrieval fee is **$0.03 per GB**. Has a **90-day minimum storage duration** and a **128 KB minimum object size**.
*   **S3 Glacier Flexible Retrieval (formerly Glacier):** Billed at **$0.0036 per GB-month**. Retrieval times range from 1 minute (expedited) to 3-5 hours (standard). Retrieval fee is **$0.03 per GB** (standard/expedited). Has a **90-day minimum storage duration** and a **40 KB minimum object size**.
*   **S3 Glacier Deep Archive:** Billed at **$0.00099 per GB-month** (~$1.01 per TB). Retrieval takes 12 hours. Retrieval fee is **$0.09 per GB** (standard). Has a **180-day minimum storage duration** and a **40 KB minimum object size**.
*   **S3 Express One Zone (Directory Buckets):** High performance, single-AZ directory storage designed for ML training and analytics. Billed at **$0.16 per GB-month** plus request execution capacity fees.

### B. Request Pricing (us-east-1)
API requests are charged based on the type of operation:
*   **Class A Requests (Write/List):** `PUT`, `COPY`, `POST`, `LIST` requests are charged at **$0.005 per 1,000 requests** (Standard).
*   **Class B Requests (Read):** `GET`, `SELECT`, and other read requests are charged at **$0.0004 per 1,000 requests** (Standard).
*   *Note: Request costs increase significantly for archival classes (e.g., PUT requests to Glacier Deep Archive are $0.05 per 1,000).*

### C. Data Retrieval Fees
When reading data from Infrequent Access or Glacier classes, you pay a per-GB fee:
*   Standard-IA / One Zone-IA: **$0.01 per GB**
*   Glacier Instant / Flexible Retrieval: **$0.03 per GB**
*   Glacier Deep Archive: **$0.09 per GB**
*   *Warning: Reading 1 TB of data from Glacier Deep Archive costs $90.00 in retrieval fees alone.*

### D. Minimum Storage Durations (Early Deletion Fees)
If you delete or overwrite an object before its class minimum duration has passed, you are billed for the remaining days as if the object was still stored:
*   Standard-IA / One Zone-IA: **30 days**
*   Glacier Instant / Flexible Retrieval: **90 days**
*   Glacier Deep Archive: **180 days**

### E. S3 Replication Costs (CRR/SRR)
Configuring Cross-Region Replication (CRR) or Same-Region Replication (SRR) incurs compounding charges:
*   **Double Storage:** Billed for the storage volume in both the source and destination buckets at their respective class rates.
*   **Replication API Charges:** Billed for the read/GET requests from the source bucket and the write/PUT requests in the destination bucket ($0.005/1K standard) to perform the copy.
*   **Egress Network Fees:** Replicating data across regions (CRR) incurs standard inter-region data transfer fees ($0.01 to $0.02 per GB) for all bytes moved.

### F. S3 Object Lambda Pricing
Allows adding custom code to S3 GET requests to modify data on the fly.
*   **Processing Cost:** Charged **$0.005 per GB** of data processed by the Lambda function.
*   **Execution Cost:** Billed standard AWS Lambda request and duration fees for the underlying function run.
*   **Standard S3 Request Fees:** Billed the standard S3 GET request fee ($0.0004 per 1,000 requests) for retrieving the base object.

*   **Requests:** 20,000 GET requests and 2,000 PUT/COPY/POST/LIST requests per month for the first 12 months.

## 6. Common Cost Hotspots & Pitfalls
*   **Object Tagging Fee Storms:** AWS charges **$0.01 per 10,000 tags** per month. If you have a bucket with 50 million small objects and put 3 tags on each object, the tagging fee alone is **$150.00/month**, which is vastly more than the storage cost of the objects!
*   **Transitioning Small Files (<128 KB) to Infrequent Access:** S3 Standard-IA has a minimum billable size of 128 KB. If you transition a 10 KB file to IA, you are billed for 128 KB of storage. This increases your cost by 12x rather than saving money.
*   **Glacier Early Deletion Costs:** Transitioning files to Glacier Deep Archive that are temporary or overwritten every 30 days. You will be billed for 180 days of storage for every deletion/overwrite.
*   **High Lifecycle Transition Request Charges:** Transitioning millions of files to Glacier using lifecycle rules. Transitioning costs **$0.05 per 1,000 objects** to Glacier Deep Archive. Transitioning 10 million objects will cost **$500.00 in API request fees** in a single day.
*   **Incomplete Multipart Uploads:** If an application upload to S3 fails halfway through, S3 retains the uploaded parts in storage. These partial files are invisible in the standard console but generate standard storage fees indefinitely.
*   **S3 Replication Egress Storms:** Replicating large buckets containing multi-terabyte objects across regions. Standard inter-region transfer charges apply to all replicated data, which can result in thousands of dollars in egress fees.
*   **Object Lambda High Processing Fees:** Using Object Lambda on large datasets (e.g., transforming gigabyte-scale CSVs or video streams on every GET request). Processing charges of $0.005/GB, combined with Lambda execution runtime fees, can quickly build up massive bills compared to static downloads.

## 7. Actionable Cost Optimization Strategies
1.  **Configure Incomplete Multipart Upload Lifecycle Rules:** Add a bucket lifecycle rule to automatically abort and clean up incomplete multipart uploads after 7 days. This is an instant, zero-risk storage cost reduction.
2.  **Enforce Object Size Constraints in Lifecycle Rules:** When creating lifecycle rules to transition data to Standard-IA or Glacier, always configure the **minimum object size filter** to `128 KB` (or larger). This prevents small files from being charged early deletion and minimum size premiums.
3.  **Use S3 Intelligent-Tiering for Dynamic Workloads:** For buckets where you cannot predict the access patterns, use **S3 Intelligent-Tiering**. It handles transitions automatically without retrieval fees. 
    *   *Warning:* Avoid Intelligent-Tiering for buckets containing millions of files under 128 KB, as the $0.0025/1,000 object monitoring fee will outweigh any storage savings.
4.  **Replace Object Tags with Metadata or Prefixes:** Instead of using S3 Object Tags for classification (which are billed), use **User Metadata** (added during upload, free) or structure your files using **prefixes (folders)** to organize data.
5.  **Consolidate Small Files (Tar/Zip) Before Archival:** If you have billions of small logs or files to archive, zip/tar them into larger chunks (e.g., 100 MB or 1 GB) before uploading to S3 Glacier. This reduces your total object count, drastically reducing both transition API fees and retrieval request overhead.
6.  **Enable Storage Lens Free Tier:** Review S3 Storage Lens dashboards weekly. It automatically highlights your top cost-saving opportunities, such as buckets with high volume of non-current versions or incomplete uploads.
